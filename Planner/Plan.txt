  SECTION 1 — PROJECT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  NAME         : Sandhya CRM
  PURPOSE      : Full internal management system for a software company.
                 Tracks customers, software products, subscriptions, invoices,
                 alerts, support tickets, communications, and team activity.
  USERS        : SuperAdmin, Admin, Standard (role-based access)
  BACKEND PORT : 5000
  FRONTEND PORT: 3000
  DATABASE     : MongoDB (local or Atlas)
  AUTH         : JWT (access token 15min + refresh token 7d)
  SMS PROVIDER : MSG91 or Twilio (WhatsApp + SMS)
  FILE STORAGE : Cloudinary or local /uploads
  CRON JOBS    : node-cron (runs renewal checks, uptime pings, alert generation)

  ── AUTH ────────────────────────────────────────────────────────────────────

  [1] /login
      - Email + Password fields
      - JWT stored in httpOnly cookie
      - Redirect to /dashboard on success
      - "Forgot password" link (sends reset email)

  [2] /forgot-password
      - Enter email → receive reset link

  [3] /reset-password/:token
      - New password + confirm password

  ── DASHBOARD ───────────────────────────────────────────────────────────────

  [4] /dashboard
      Widgets row 1 (KPI cards):
        - Total Active Customers
        - Monthly Revenue (current month invoices paid)
        - Pending Dues (subscriptions overdue)
        - Open Support Tickets
        - Softwares Live / Broken count

      Widgets row 2 (alerts preview):
        - Upcoming Renewals (next 7 days) — list with customer name, software, days left
        - Infrastructure Expiries — domain/hosting/SSL due in 30 days
        - Broken Software URLs — red badge count

      Widgets row 3 (activity):
        - Recent Communications (last 5 SMS/calls)
        - Recent Invoices (last 5 with status badge)
        - Quick Actions: [Add Customer] [Add Software] [Create Invoice] [Send Alert]

      Role visibility:
        - SuperAdmin: sees all widgets including team activity
        - Admin: sees all except audit logs
        - Standard: sees only their assigned customers

  ── CUSTOMERS ───────────────────────────────────────────────────────────────

  [5] /customers
      - Data table: Name, Phone, Business, Softwares count, Status badge,
                    Renewal status, Assigned to, Actions
      - Filters: Status (Active/Expired/Lead/Suspended), Source, Priority,
                 Tag, Assigned User, Renewal this month
      - Search: by name, phone, email, business name
      - Sort: by name, renewal date, created date
      - Pagination: 20 per page
      - Export to CSV button (permission: reports.export)
      - [+ Add Customer] button

  [6] /customers/new
      Form fields:
        - Name (required)
        - Email (required, unique)
        - Phone (required)
        - WhatsApp (optional, defaults to phone)
        - Business Name
        - City, State, Country
        - Source (dropdown: Referral/Instagram/Facebook/Google/WalkIn/Other)
        - Priority (dropdown: Normal/High/VIP)
        - GST Number
        - Tags (multi-select, searchable)
        - Assigned Staff (dropdown of Users)
        - Notes (textarea)
        - Document upload (agreement/PO) — max 5 files
      On submit: creates Customer + writes AuditLog

  [7] /customers/:id
      Customer profile page. Tabs:

      TAB 1 — Overview
        - All fields from customer record (editable inline)
        - Active subscriptions list with renewal dates + status
        - Quick stats: total paid, total due, tickets open

      TAB 2 — Subscriptions
        - List of all subscriptions (active, expired, cancelled)
        - Per subscription: Software name, buy date, renewal date,
          amount, billing cycle, payment status, reminderSent flags
        - [+ Add Subscription] button → opens modal

      TAB 3 — Invoices
        - List of all invoices for this customer
        - Per invoice: invoice#, software, amount, status, date
        - [View PDF] button
        - [Mark as Paid] button (permission: invoices.markPaid)
        - [+ Generate Invoice] button

      TAB 4 — Communications
        - Chronological timeline of all SMS/WhatsApp/calls
        - Each entry: channel icon, direction, message, delivery status, date
        - [+ Log Communication] button (manual entry)
        - [Send SMS/WhatsApp] button → opens message composer

      TAB 5 — Support Tickets
        - List of tickets raised by this customer
        - Per ticket: TKT#, title, software, priority, status, assigned to, date
        - [+ New Ticket] button

      TAB 6 — Activity Log
        - Full AuditLog for this customer record
        - Who changed what, when, before/after values

         - All support tickets for this software

       TAB 5 — Activity Log
         - AuditLog entries for this software record

  SECTION 4 — ALL API ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  BASE URL: /api/v1

  ── AUTH ─────────────────────────────────────────────────────────────────────
  POST   /auth/login                     Login, returns JWT + refresh token
  POST   /auth/logout                    Invalidate refresh token
  POST   /auth/refresh                   Issue new access token using refresh
  POST   /auth/forgot-password           Send reset link to email
  POST   /auth/reset-password/:token     Set new password
  GET    /auth/me                        Get logged-in user profile

  ── USERS / TEAM ─────────────────────────────────────────────────────────────
  GET    /users                          List all users (SuperAdmin)
  POST   /users                          Create user (SuperAdmin)
  GET    /users/:id                      Get user profile
  PUT    /users/:id                      Update user (SuperAdmin)
  PATCH  /users/:id/permissions          Update permissions (SuperAdmin)
  PATCH  /users/:id/toggle-active        Activate/deactivate user
  DELETE /users/:id                      Delete user (SuperAdmin)

  ── CUSTOMERS ────────────────────────────────────────────────────────────────
  GET    /customers                      List with filters + pagination
  POST   /customers                      Create customer
  GET    /customers/:id                  Get full customer profile
  PUT    /customers/:id                  Update customer
  DELETE /customers/:id                  Delete customer (SuperAdmin only)
  GET    /customers/:id/subscriptions    All subscriptions for customer
  GET    /customers/:id/invoices         All invoices for customer
  GET    /customers/:id/communications   Communication timeline
  GET    /customers/:id/tickets          Support tickets for customer
  GET    /customers/:id/activity         AuditLog for customer record
  GET    /customers/export/csv           Export customers as CSV

  ── SOFTWARES ────────────────────────────────────────────────────────────────
  GET    /softwares                      List with filters + pagination
  POST   /softwares                      Create software
  GET    /softwares/:id                  Get software detail
  PUT    /softwares/:id                  Update software
  DELETE /softwares/:id                  Delete (SuperAdmin, only if no active subscriptions)
  GET    /softwares/:id/customers        Customers subscribed to this software
  GET    /softwares/:id/tickets          Tickets for this software
  POST   /softwares/:id/ping             Manual uptime check trigger
  GET    /softwares/:id/uptime-log       Last 50 uptime check results

  ── SUBSCRIPTIONS ────────────────────────────────────────────────────────────
  GET    /subscriptions                  List all with filters + pagination
  POST   /subscriptions                  Create subscription
  GET    /subscriptions/:id             Get subscription detail
  PUT    /subscriptions/:id             Update subscription
  DELETE /subscriptions/:id             Cancel/delete subscription
  POST   /subscriptions/:id/renew       Renew subscription (updates dates + creates invoice)

  ── INVOICES ─────────────────────────────────────────────────────────────────
  GET    /invoices                       List with filters + pagination
  POST   /invoices                       Generate invoice
  GET    /invoices/:id                   Get invoice detail

  ── CRON JOBS ────────────────────────────────────────────────────────────────

  renewal.job.js     → runs: every day at 8:00 AM
    - Query subscriptions where renewalDate is 30, 7, or 1 days from today
    - For each match: create Alert if not already created (check reminderSent flags)
    - Set reminderSent.thirtyDays / sevenDays / oneDay = true
    - Create Notification for assigned staff
    - Query subscriptions where renewalDate < today + gracePeriodDays
    - Set status = Expired, create PaymentOverdue Alert

  infra.job.js       → runs: every day at 9:00 AM
    - Query softwares where hostingExpiryDate is within 30 days
    - Query softwares where domainExpiryDate is within 30 days
    - Query softwares where sslExpiryDate is within 30 days
    - For each: create Infra Alert if not already pending
    - Assign severity: 30d = Info, 14d = Warning, 7d = Urgent

  uptime.job.js      → runs: every 15 minutes
    - Call uptime.service.checkAllSoftwares()
    - If software goes from Up → Down: create SoftwareBroken Alert
    - If software comes back Up: resolve previous Down alert
    - Update uptimeStatus, lastCheckedAt, lastDownAt on Software
    - Calculate rolling 30-day uptimePercent

  invoice.job.js     → runs: every day at 7:00 AM
    - Query invoices where paymentStatus = Pending AND dueDate < today
    - Set paymentStatus = Overdue
    - Create PaymentOverdue Alert if not exists
    - Set reminderSent = false so next cycle re-notifies

  Every request flows through:
    1. express.json() + express.urlencoded()
    2. cors() — allow frontend origin
    3. helmet() — security headers
    4. morgan() — request logging
    5. auth.middleware (verifyJWT) — all /api/v1 routes except /auth/*
    6. role.permissions — route-level role check (e.g. SuperAdmin only)
    7. permission.middleware — granular check (e.g. customers.edit)
    8. Controller logic
    9. audit.middleware — auto-log POST/PUT/PATCH/DELETE to AuditLog
    10. response.util — standard JSON shape { success, data, message, pagination }

  Error handler (global):
    - Mongoose CastError → 400
    - Mongoose ValidationError → 400 with field errors
    - Duplicate key (code 11000) → 409 "already exists"
    - JWT errors → 401
    - Permission errors → 403
    - Not found → 404
    - All others → 500

  SECTION 12 — KEY BUSINESS RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. A customer can have multiple subscriptions (one per software).
  2. A subscription can only be deleted if it has no paid invoices.
  3. A software cannot be deleted if it has active subscriptions.
  4. Invoice numbers are sequential and never reused (even on delete).
  5. Audit logs are append-only — no update or delete allowed.
  6. SMS reminders are gated by reminderSent flags — never send twice.
  7. When a subscription is renewed, create a new Invoice automatically.
  8. If software URL returns non-200 for 2 consecutive pings → status = Broken.
  9. Alerts auto-resolve when the underlying condition is fixed
     (e.g. invoice paid → close PaymentOverdue alert).
  10. Standard users only see data for customers assigned to them (serviceUser).
  11. SuperAdmin can see and do everything — permissions array is ignored.
  12. Passwords must be bcrypt hashed — never stored plain.
  13. Refresh tokens are stored in httpOnly cookies — not localStorage.
  14. All file uploads (documents, invoices) go to Cloudinary — no local storage.

  - Hover state on clickable cards    : border color changes to #1a73e8
    (same as input focus color on Login)

  ── BORDER RADIUS ────────────────────────────────────────────────────────────

  - Cards / panels / containers : borderRadius: 8px
  - Buttons                     : borderRadius: 6px
  - Inputs                      : borderRadius: 6px
  - Badges / pills              : borderRadius: 4px
  - Avatar circles              : borderRadius: 50%
  - Icon containers (KPI)       : borderRadius: 7px
  - No 14px / 16px rounded cards — keep it tight and business-like

  ── TYPOGRAPHY ───────────────────────────────────────────────────────────────

  - Font family   : 'Inter', system-ui, -apple-system, 'Segoe UI', sans-serif
  - Page title    : 18px, fontWeight 700, color #111827
  - Section title : 13.5px, fontWeight 600, color #111827
  - Body / table  : 13px, color #111827
  - Secondary text: 12px, color #9ca3af
  - Section label : 11px, fontWeight 600, color #9ca3af, textTransform uppercase,
                    letterSpacing 0.5px
  - KPI value     : 26px, fontWeight 700, letterSpacing -0.5px, color #111827
  - No large decorative headings on inner pages

  ── SPACING — COMPACT, NOT AIRY ──────────────────────────────────────────────

  - Page top margin between title and content : 20px
  - Gap between grid cards                    : 12px
  - Gap between grid rows                     : 12–16px
  - Panel header padding                      : 13px 18px
  - Panel body padding                        : 14px 18px
  - Table cell padding                        : 10px 16px (TD), 8px 16px (TH)
  - KPI card inner padding                    : 16px
  - Quick-tile / module-tile padding          : 16px
  - No 28px / 40px / 52px padding inside content cards

  ── COLORS ───────────────────────────────────────────────────────────────────

  - Primary blue    : #1a73e8 (buttons, links, active states, icon highlights)
  - Hover blue      : #1557b0 (primary button hover)
  - Primary text    : #111827
  - Secondary text  : #374151
  - Muted text      : #6b7280
  - Placeholder     : #9ca3af
  - Border          : gainsboro (= #dcdcdc)
  - Row border      : #f3f4f6
  - Error red       : #dc2626 / bg #fef2f2
  - Success green   : #16a34a / bg #f0fdf4
  - Warning amber   : #d97706 / bg #fffbeb
  - Info teal       : #0891b2

  ── BADGES / PILLS ───────────────────────────────────────────────────────────

  - Shape   : borderRadius 4px (rectangular, not round)
  - Size    : fontSize 11px, fontWeight 600, padding 2px 8px
  - Colors follow the semantic color palette above (status, action, severity)
  - No circular dot badges on status — use rectangular pills

  ── INTERACTIVE STATES ───────────────────────────────────────────────────────

  - Clickable card hover : border-color → #1a73e8, no transform/shadow changes
  - Button hover         : background darkens (e.g. #1a73e8 → #1557b0)
  - Table row hover      : background → #f9fafb
  - Link hover           : text-decoration underline OR background → #eff6ff pill
  - Input focus          : border-color → #1a73e8 (no glow/shadow)
  - No translateY / scale transforms on hover — keep it flat and stable

  ── WHAT NOT TO DO ───────────────────────────────────────────────────────────

  - No blue/purple/gradient welcome banners
  - No box-shadow as primary card differentiation
  - No large borderRadius (14px+) on content cards
  - No excessive margin/padding that makes the UI feel "airy" or "landing-page"
  - No colored card backgrounds (eff6ff, f0fdf4 fills on entire cards)
  - No drop shadows on buttons — use background color change instead
  - No emojis in UI labels or headings
  - No animation/transition other than border-color and background-color changes
